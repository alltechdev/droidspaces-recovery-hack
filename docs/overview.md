# Project Overview

## What This Project Is

This repository is a **hacked Android recovery image** for a Samsung device running a MediaTek MT6835 SoC. The goal is to transform the stock Samsung recovery partition into a fully-functional Linux environment that can boot an Ubuntu 24.04 container using the **Droidspaces** container runtime — all without modifying the main Android system partition.

The project uses the [Android_boot_image_editor](https://github.com/cfig/Android_boot_image_editor) toolchain (based on Gradle/Kotlin) to unpack, modify, and repack the `recovery.img` binary.

## Target Device

- **SoC**: MediaTek MT6835 (Dimensity 6100+)
- **Manufacturer**: Samsung
- **Kernel**: Linux 5.15.148 (ARM64), compiled with Android LLVM/Clang 14.0.7
- **Recovery format**: Android boot image with AVB (Android Verified Boot) hash footer
- **Stock recovery binary**: `recovery.img` (80 MiB)

## What Was Changed

At a high level, the recovery ramdisk was modified to:

1. **Root the ADB shell** — `adbd` starts as root immediately after boot, giving full shell access over USB.
2. **Disable SELinux enforcement** — A custom `selinux-permissive` service runs at `init` time and writes `0` to `/sys/fs/selinux/enforce`, putting the entire kernel in permissive mode.
3. **Patch the SELinux policy binary** — The precompiled `sepolicy` blob was patched with `magiskpolicy` to allow `adbd` → `su` domain transitions and mark `adbd`, `su`, and `recovery` domains as permissive.
4. **Replace the kernel** — The stock Samsung kernel was replaced with a custom build that has Droidspaces container runtime support compiled in.
5. **Suppress the stock recovery UI** — The `recovery` service is marked `disabled` so the Samsung recovery menu never appears.
6. **Add the `recovery-console` display server** — A binary called `recovery-console` drives the device display and is used as the output sink when running the Ubuntu container.
7. **Add the Droidspaces daemon and container launch scripts** — The `droidspaces` binary runs as a system daemon; `boot-ubuntu.sh` launches an Ubuntu 24.04 rootfs image.
8. **Wire up hardware drivers** — MTK gen4m Wi-Fi and FocalTech FT3419U touchscreen kernel modules are loaded during recovery boot.
9. **Centralize USB ConfigFS init logic** — All ADB/fastboot USB gadget setup was moved to a dedicated `init.recovery.usb.rc` file.

## Repository Layout

```
.
├── recovery.img                       # Stock recovery image (starting point)
├── build/
│   └── unzip_boot/                    # Unpacked recovery image (working directory)
│       ├── kernel                     # Kernel image (replaced with custom build)
│       ├── dtb                        # Device-tree blob (stock)
│       ├── ramdisk.img                # Packed ramdisk (generated by Gradle; gitignored, not in repo)
│       ├── recovery.json              # Boot image metadata (header, offsets, etc.)
│       ├── recovery.avb.json          # AVB footer metadata
│       ├── kernel_version.txt         # Extracted kernel version string
│       ├── kernel_configs.txt         # Full kernel .config extracted from kernel image
│       └── root/                      # Ramdisk root filesystem (all modifications here)
│           ├── system/bin/            # Executables: adbd, recovery, recovery-console,
│           │                          #   droidspaces, boot-ubuntu.sh, selinux-permissive,
│           │                          #   wlan_init.sh, sh, toybox, toolbox, …
│           ├── system/etc/init/       # Android init .rc files
│           │   ├── hw/init.rc         # Primary init script (heavily modified)
│           │   ├── init.recovery.usb.rc        # USB gadget configuration
│           │   ├── init.mtk.wlan.rc            # Wi-Fi driver init
│           │   ├── init.touch.rc               # Touch driver init
│           │   └── init.ubuntu-droidspaces.example.rc  # Container launch services
│           ├── system/etc/environment # Shell environment (PS1 prompt)
│           ├── system/lib64/          # Shared libraries
│           ├── lib/modules/           # Kernel modules (system partition)
│           ├── vendor/lib/modules/    # Vendor kernel modules (touch, sec_cmd, tuihw)
│           ├── vendor/wlan_modules/   # MTK Wi-Fi kernel module stack
│           ├── vendor/firmware/       # Firmware blobs (Wi-Fi, BT, touch calibration)
│           ├── vendor/bin/            # Vendor binaries (wmt_loader, wmt_launcher)
│           ├── sepolicy               # Patched SELinux policy binary
│           ├── plat_property_contexts # Property contexts (ro.force.debuggable added)
│           └── prop.default           # Default properties (ro.secure=0, debuggable=1)
├── aosp/                              # AOSP tools (avbtool, mkbootimg, signing keys, …)
├── bbootimg/                          # Boot image editor JAR
├── src/                               # Gradle source / integration test resources
├── tools/                             # Helper scripts and diffs
├── gradlew / gradlew.bat              # Gradle wrapper (entry point for pack/unpack)
└── docs/                              # This documentation
```

## Workflow

```bash
# 1. Start from the stock image already in the repo
#    recovery.img is committed at the root

# 2. Unpack (already done; unpacked files are in build/unzip_boot/)
./gradlew unpack

# 3. Make modifications to build/unzip_boot/root/

# 4. Repack — produces recovery.img.signed
./gradlew pack

# 5. Flash to device via fastboot / Odin / heimdall
fastboot flash recovery recovery.img.signed
```
